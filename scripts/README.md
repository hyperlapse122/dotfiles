# scripts/

Bootstrap helpers plus manual auth, package, and system setup scripts.

## Layout

| Directory | Purpose |
|---|---|
| `scripts/auth/` | Manual authentication helpers and CLI auth configuration shared by bootstrap |
| `scripts/bootstrap/` | Cross-platform helpers invoked by dotbot bootstrap steps |
| `scripts/linux/` | Linux-only system, package, and desktop setup helpers |

## Current scripts

| Script | Platform | Called by | Purpose |
|---|---|---|---|
| `auth/auth-gh.sh` / `auth/auth-gh.ps1` | macOS+Linux / Windows | Manual | Starts GitHub CLI web login for `github.com`, preferring system `gh` and falling back to `mise exec gh@latest -- gh` |
| `auth/auth-glab.sh` / `auth/auth-glab.ps1` | macOS+Linux / Windows | Manual | Starts GitLab CLI web login for `git.jpi.app` and `gitlab.com`, preferring system `glab` and falling back to `mise exec glab@latest -- glab` |
| `auth/auth-tailscale.sh` | Linux only | Manual | Runs `tailscale up --operator "$USER" --accept-routes`, using `sudo` when not already root. Single-platform by design because it targets the Linux Tailscale service configured by `scripts/linux/install-packages.sh` |

| `auth/import-gpg-keys.sh` / `auth/import-gpg-keys.ps1` | macOS+Linux / Windows | Manual | Reads the private GPG key from 1Password via `op read` and imports it with `gpg --batch --import` |
| `bootstrap/inject-1password-secrets.sh` / `bootstrap/inject-1password-secrets.ps1` | macOS+Linux / Windows | `install.<os>.yaml` `shell:` step (also runnable manually) | Finds every `*.1password` template in the repo, renders it with `op inject`, and writes it under `~/.secrets/<file-name-without-.1password>`; no-ops when no templates exist; secret files are mode `0600` on Unix, and secret directories are owner-only/traversable (`0700`) so the files remain readable by the owner |
| `bootstrap/install-fonts.sh` / `bootstrap/install-fonts.ps1` | macOS+Linux / Windows | `install.<os>.yaml` `shell:` step (also runnable manually) | Installs desktop fonts user-wide from GitHub Releases. Skips already-installed fonts unless `--force` / `-Force`. Add fonts via the registry block at the top of each script |
| `bootstrap/install-prezto.sh` | macOS+Linux | Manual | One-time Prezto clone into `${ZDOTDIR}/.zprezto`; zsh-only so no `.ps1` (single-platform by design); fails fast when already cloned |
| `bootstrap/render-opencode-prompt-append.sh` / `bootstrap/render-opencode-prompt-append.ps1` | macOS+Linux / Windows | `install.conf.yaml` `shell:` step (also runnable manually) | Uses mise-managed Node.js to render `home/.config/opencode/prompts/*_prompt_append.md` into the matching `prompt_append` values in `home/.config/opencode/oh-my-openagent.jsonc`; shared logic lives in `bootstrap/render-opencode-prompt-append.mjs` |
| `bootstrap/configure-codex-config.sh` / `bootstrap/configure-codex-config.ps1` | macOS+Linux / Windows | `install.conf.yaml` `shell:` step (also runnable manually) | Uses mise-managed Node.js to merge the repo-tracked shared Codex settings in the top-level `codex/codex-config.managed.toml` into the machine-local `$CODEX_HOME/config.toml` (default `~/.codex/config.toml`), updating/inserting only the declared keys and preserving everything else — most importantly the machine-local `[projects]` trust table Codex writes back into that file (which is why it is rendered, not dotbot-symlinked: Codex has no include directive and would otherwise mutate the repo). Idempotent; aborts without writing rather than risk corrupting an ambiguous multi-line value; backs up to `config.toml.bak` before changing; no-op when the managed file has no active keys (the default). Shared logic lives in `bootstrap/configure-codex-config.mjs`. Flags: `--check`, `--print`, `--no-backup` |
| `linux/install-linux-system-config.sh` | Linux only | `install.linux.yaml` `shell:` step | Recursively discovers files in `system/linux/etc/` and runs `sudo install -D -m 644` to their absolute paths. The `etc/sudoers.d/*` subtree is special-cased: installed at mode `0440` and only on virtual machines (gated on `systemd-detect-virt --vm`); contents are syntax-checked with `visudo -c -f` before install. The ThinkPad `thinkpad_acpi` module-load/modprobe drop-ins are also special-cased: installed only when `dmidecode -t system` reports a ThinkPad, then `modprobe thinkpad_acpi` is run so `fan_control=1` is active without waiting for the next boot. After the file-install loop, iterates the hard-coded `REMOVED_ETC_PATHS=(…)` manifest at the top of the cleanup section and `rm -f`s each entry that still exists (handles regular files and dangling symlinks). Contributors who delete a tracked file under `system/linux/etc/` add its absolute `/etc` path to that array in the same commit; the manifest propagates the cleanup to every machine after pull. Then runs `systemctl daemon-reload` when `system/linux/etc/systemd/system/` exists. The `docker-prune.service` and `docker-prune.timer` units are now listed in `REMOVED_ETC_PATHS` so machines that previously enabled the docker-era weekly-prune timer get the orphan units removed on pull (Podman handles rootless image cleanup from the user session instead, so the timer is no longer re-enabled). Enables lingering for the invoking user with `loginctl enable-linger` so rootless Podman `systemd --user` units survive logout and start at boot, then masks the system-level rootful API with `systemctl mask podman.socket podman.service` (sudo `podman` is daemonless, so the mask only blocks the rootful socket path). Finally configures firewalld for Tailscale and VMware: enables IPv4 masquerade on the default zone (Tailscale exit-node and VMware NAT egress paths), binds `tailscale0` to the `trusted` zone, and opens UDP 41641 (WireGuard) and UDP 3478 (STUN) on the `public` zone. Every change is gated on a `--query-*` probe so it only mutates when something would change, with a single `--reload` at the end. The whole block is gated on `firewall-cmd --state` so it skips cleanly when firewalld is not the active backend. Exits 0 immediately when invoked from a non-interactive shell (no TTY on stdin) without cached sudo credentials — agent / CI runs where sudo would hang are skipped cleanly, and the user is told to re-run manually |
| `linux/install-packages.sh` | Linux only (Fedora-oriented) | Manual | On desktop-class chassis only, configures Fedora repo metalinks to request only KR mirrors from MirrorManager; enables Fedora third-party repos, keyd and mise COPRs, RPM Fusion, 1Password, VSCodium, Tailscale, and Proton VPN repos (Docker CE is intentionally not enabled — the Podman ecosystem covers the same workflows rootless); installs `clang21-libs` before Proton VPN so the split-tunneling daemon can resolve `libclang-cpp.so.21.1` during package post-install; installs development tools plus CLI, fcitx5, keyd, .NET, mise, Solaar, lm_sensors, 1Password, VSCodium, Chromium (Fedora base repo, not Google Chrome — only the Chromium build honors the user-scope `~/.config/chromium/External Extensions/` auto-install mechanism), Tailscale, Proton VPN, and VirtualBox (with `akmod-VirtualBox` and `kernel-devel`) packages; adds Steam/Discord only on bare metal; installs selected dotnet global tools; downloads and installs the VirtualBox Extension Pack matching the installed VirtualBox version (SHA256-verified against the upstream `SHA256SUMS` manifest, license auto-accepted from the bundled `ExtPack-license.txt` hash, `--replace` keeps it idempotent across upgrades); installs the rootless Podman stack (`podman`, `podman-docker` for the `docker` CLI shim, `podman-compose`, `buildah`, `containers-common`, `passt`, `fuse-overlayfs`, `slirp4netns`) and removes the user from a leftover `docker` group when one still exists from a prior Docker install; allocates subordinate UID/GID ranges (`/etc/subuid`, `/etc/subgid`, 100000-165535) for rootless user namespaces, guarded so existing entries are left alone; enables `keyd`, `tailscaled`, and `libvirtd`; enables `vboxdrv` for next boot; runs `akmods` and, on UEFI hosts with Secure Boot enabled, batches unenrolled `/etc/pki/akmods/certs/*.der` keys into a single `mokutil --import`; adds the user to `keyd`, `libvirt`, and `vboxusers` groups |
| `linux/config-kde.sh` | Linux only (KDE Plasma 6) | `install.linux.yaml` `shell:` step (also runnable manually) | Configures KDE Plasma 6 user-side settings for fonts, touchpad natural scrolling/tap-to-click/clickfinger, panel grouping, Kickoff list view, Digital Clock calendar/date, Fcitx virtual keyboard, disabled screen-edge/corner behavior, a Spotlight-style KRunner (centered free-floating box + history autocompletion), the kill runner's trigger word ("kill") + CPU-usage sorting, and Dolphin opening the home folder on startup. Requires fonts from `bootstrap/install-fonts.sh` and skips cleanly when a suitable KDE session is not available |
| `linux/enable-fingerprint.sh` | Linux only (Fedora) | Manual | Installs fprintd and enables the authselect `with-fingerprint` feature; finger enrollment is done afterwards by the user |
| `linux/setup-luks-tpm2-unlock.sh` | Linux only | Manual | Sets up TPM2-backed automated unlocking for every `crypto_LUKS` device (the one backing `/` included) on the currently-running system: per device it enrolls a TPM2 token with `systemd-cryptenroll --tpm2-device=auto --tpm2-pcrs=7` (PCR 7 = the UEFI Secure Boot policy state; passed explicitly because systemd v258+ defaults to no PCR binding), prompting INTERACTIVELY for an existing LUKS passphrase to authorize the change — the passphrase is never read from argv/env/file (no `$PASSWORD`/`$NEWPASSWORD`, no `--unlock-key-file`, no `cryptenroll.passphrase` credential). It then merges `tpm2-device=auto` (plus `x-initrd.attach` for the root-backing device) into `/etc/crypttab` without duplicating options, drops `/etc/dracut.conf.d/tpm2-tss.conf`, and rebuilds the initramfs (`dracut -f --regenerate-all`). Enrollment ADDS a keyslot, so the existing passphrase keeps working as a fallback; `--recovery-key` enrolls an extra printed recovery key. Idempotent (skips devices that already hold a `systemd-tpm2` token; merges crypttab options; backs up `/etc/crypttab` before the first edit). Requires a TTY (it must prompt) so it is intentionally NOT wired into `install.linux.yaml`. Flags: `--dry-run` (preview, mutates nothing), `--recovery-key`, `--tpm2-pcrs SPEC` (default `7`), `-h`/`--help`. Single-platform by design (LUKS/TPM2/dracut are Linux-only) so it has no `.ps1` counterpart |

## Conventions

- Scripts that apply to macOS/Linux and Windows ship in `.sh` and `.ps1` pairs with equivalent behavior.
- Single-platform scripts are allowed only when the platform dependency is intrinsic; note that reason in the script header and in this README.
- Bootstrap-invoked paths in `install.conf.yaml` and `install.<os>.yaml` must stay in sync with this directory layout.
