## Findings considered and rejected

Recorded so they are not re-audited next run:

- **opencode secret-path "mismatch"** — REJECTED (false positive). An audit pass
  claimed `readonly_opencode.json`'s `{file:./secrets/*}` references had no
  matching rendered directory because the source dir is
  `dot_config/opencode/exact_private_secrets/`. chezmoi strips the `exact_` and
  `private_` **directory attribute** prefixes, so it renders to
  `~/.config/opencode/secrets/` — which matches the config exactly. No bug.
- **luks `${root_src%%[*}` "syntax error"** — REJECTED (false positive). In
  `dot_local/bin/executable_setup-luks-tpm2-unlock.sh:398` this is valid,
  intentional bash that strips a btrfs `[subvol]` suffix from `findmnt` output
  (see the comment on line 395). Not malformed.
- **`ts-pattern` "not bundled" into the haptic plugin** — REJECTED. It is listed
  in `packages/opencode-mxm4-haptic/tsdown.config.ts` under `alwaysBundle`, so
  the built `dist/index.mjs` chezmoi symlinks into `~/.config/opencode/plugins/`
  is self-contained.
- **TS packages "under-tested / vacuous tests"** — REJECTED. The three packages
  have rigorous `node --test` suites: real Unix-socket servers (not mocks),
  meaningful assertions, an explicit characterization test, regression guards
  citing prior plans, and a drift-guard that parses the Rust source for
  Rust↔TS waveform-table parity. (The one real gap — the Rust notify bridge's
  D-Bus parsers — is plan 005.)
- **Blanket `latest` toolchain pins in `dot_config/mise/config.toml`** —
  CONSIDERED, not worth a plan. It is a deliberate reproducibility trade-off for
  a personal dotfiles repo, gated by a documented `minimum_release_age` cooldown
  + excludes list. Pinning core tools to exact versions would add churn for
  little benefit here. Left as the maintainer's standing choice.
